## 1. 项目是什么

- **名称**：WytaoF Browser（化名 / 公开发布名）
- **类型**：Wails 桌面应用（Go 后端 + Vue 3 前端 + 内嵌 Chromium 内核）
- **当前版本**：`1.2.0`（来源：`wails.json` → `info.productVersion`）
- **作用**：本机管理多个互相隔离的指纹浏览器实例，统一管理代理池、Chrome 内核、快捷启动码、自动化接口
- **个人定位**：私有自用工具，作者署名为化名"蔚蓝海岸"，不暴露真实身份

| 产物 | 路径 | 大小 | 说明 |
|---|---|---|---|
| **Windows 安装包** | `publish\output\WytaoFBrowser-Setup-1.2.0.exe` | ~410 MB | 给非技术朋友用，双击安装 |

## 4. 打包/发布流程（Windows，最常用）

### 4.1 一键命令

```powershell
# 进入项目根目录
cd E:\1.MyWorkspace\工具\01-核心项目\WytaoF-Browser

# 交互式选择 Windows / Linux / 两者
bat\publish.bat

# 或者直接指定参数（无交互）
bat\publish.bat W                      # 仅 Windows
bat\publish.bat L                      # 仅 Linux（需 Docker Desktop）
bat\publish.bat B                      # 两者
bat\publish.bat W -Version 1.3.0       # 同时覆盖版本号
```

### 4.2 完整 Windows 流程（`bat\publish.ps1` 自动执行）

1. 解析版本号（参数 `-Version` > `wails.json productVersion`）
2. 找 NSIS：`MAKENSIS_PATH` → `NSIS_PATH` → `NSIS_HOME` → `PATH` → 默认 `C:\Program Files (x86)\NSIS\makensis.exe`
3. 校验 `publish\runtime-manifest.json` 中 windows-amd64 文件 SHA256
4. 跑 `wails build` 生成 `build\bin\wytaof-browser.exe`
5. 组 staging：
   - 复制 `build\bin\wytaof-browser.exe` → `publish\staging\`
   - 复制 **`publish\config.init.yaml`**（干净配置） → `publish\staging\config.yaml`
   - 复制 `bin\xray.exe`、`bin\sing-box.exe` → `publish\staging\bin\`
   - 复制 `chrome\<core>\` → `publish\staging\chrome\<core>\`（自动检测有效 chrome.exe）
   - 创建**空** `publish\staging\data\` 目录（**不带 app.db**，首次启动初始化）
6. 调 makensis 编译 `publish\installer.nsi` → `publish\output\WytaoFBrowser-Setup-<ver>.exe`
7. 删除 `publish\staging\`

### 4.3 只构建便携 exe（不打安装包）

```powershell
bat\build.bat
# 产物：build\bin\wytaof-browser.exe
```

### 4.4 Linux 包（在 Windows 上跑）

需要 **Docker Desktop 已启动**，跑 `bat\publish.bat L` 即可，内部会调：

```text
docker build -f publish/linux/linux-builder.Dockerfile -t wytaof-browser-linux-builder:local publish/linux
docker run --rm -v <repo>:/workspace -w /workspace wytaof-browser-linux-builder:local \
  bash -c "bash publish/linux/publish-linux.sh --arch <amd64|arm64>"
```

### 4.5 macOS unsigned 包

**必须在 macOS 主机上跑**：

```bash
bash publish/mac/publish-mac.sh --arch amd64
bash publish/mac/publish-mac.sh --arch arm64
```

## 5. 开发与本地启动

### 5.1 Windows 开发（最常用）

```bat
bat\dev.bat            # 默认稳定模式：先 build frontend/dist，再以静态资源启动 Wails
bat\dev.bat live       # 启 Vite watcher + Wails dev（HMR 联调时用）
bat\dev.bat limited    # 同 live 但叠加内存上限（低内存机或调内存膨胀时用）
```

### 5.2 重要环境变量

| 变量 | 用途 |
|---|---|
| `MAKENSIS_PATH` / `NSIS_PATH` / `NSIS_HOME` | 指定 NSIS 位置 |
| `DEV_PROXY_URL` / `DEV_NO_PROXY` / `DEV_GOPROXY` | 给 npm/Node/Go 下载流量加代理 |
| `FRONTEND_PROCESS_MEMORY_LIMIT_MB` | `bat\dev.bat limited` 的内存上限 |
| `ANT_BROWSER_PUBLISH_BEST_COMPRESSION=1` | NSIS 用 LZMA 高压缩（更小但更慢） |
| `NO_PAUSE=1` / `CI=1` | 脚本结束不 pause（CI 调用） |

### 5.3 数据目录

| 平台 | 用户数据位置 |
|---|---|
| Windows 开发 | 项目内 `data\` 和 `build\bin\data\` |
| Windows 安装版 | 安装目录下的 `data\`（NSIS 创建空目录，首启初始化） |
| macOS | `~/Library/Application Support/wytaof-browser` |
| Linux | 看 `publish/linux/README.md` |

## 7. 注意事项 / 易忘点

- ⚠️ **`publish\config.init.yaml` 才是发布版默认配置**，不是 `config.yaml` / `build\bin\config.yaml`。改错地方等于没改。
- ⚠️ **不要把 `build\bin\` 整目录发给别人**：里面 `data\app.db` + `gam_<邮箱>_*` 目录会暴露你的 Google 账号。
- ⚠️ **`installer.nsi` 必须是 UTF-8 BOM 编码**（否则 makensis 中文乱码报语法错）。`bat\publish.ps1` 在调 makensis 前会自动重写为 BOM，但手动跑 makensis 时要注意。
- ⚠️ **Linux 打包必须有 Docker Desktop 在跑**，否则 `bat\publish.bat L` 会失败。
- ⚠️ **macOS 打包必须在原生 macOS 主机**，不能在 Windows 上交叉编译。
- ⚠️ **代理运行时是仓库内置**（`bin\xray.exe` / `bin\sing-box.exe`），别再单独下；要更新走 `tools\runtime\sync-runtime.py`。
- ⚠️ **wails.json 的 author/companyName/comments**："蔚蓝海岸" + "个人自用版" — 化名安全，但 PE 资源实测为空（详见上面"维护操作"），如果分发后想让接收方看到正常版本号，需要修复 wails 嵌入流程。
- 📌 **分支约定**：`master` 是干净开发基线（不带 `data\app.db`）；`user_data` 带一份测试数据库快照用来联调演示。
- 📌 **public-release 工作流**（`tools\public-release\publish-public.ps1`）是另一条独立链路，主流程不用它，与 `bat\publish.ps1` 平行存在；具体差异看 `tools\public-release\README.md`。
- 📌 **wails 绑定刷新**：改了 Go 端导出 service / 数据结构，需要跑 `bat\generate-bindings.bat` 或让 `bat\dev.bat` 自动跑。
